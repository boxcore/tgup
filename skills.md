# 🚀 tgup — Telegram 自动化控流媒体投递工具说明书

`tgup` 是一款专为大流量媒体投递、自建频道/群组自动化运维量身定制的 Go 语言底层高精度媒体上传工具。完美支持 WebDAV/Alist/CloudDrive2 等网盘挂载环境下的 0 字节重复流量流式传输。

---

## 📦 编译与部署

### 1. 初始化依赖与配置
```bash
# 自动整理项目依赖
go mod tidy

# 创建配置与日志缓存目录
mkdir -pv ~/.tgup/{cache,logs}
```

### 2. 编译并安装
```bash
# 在项目根目录下，指定入口路径编译为可执行文件
go build -o tgup ./cmd/tgup

# 赋予执行权限并移入系统 bin 目录以便全局调用（可选）
chmod +x tgup
sudo mv tgup /usr/local/bin/
```

## ⚡ 核心硬核演进特性说明

### 💎 1. 特征码预判与元数据强缓存（0 流量损耗悖论解法）

脚本在对 >2GB 的网盘大视频处理时，摒弃了“先下载再算 SHA1”的低效做法。通过秒级提取 **`网盘路径 + 文件体积 + 修改时间戳`** 混合拼装算出唯一特征码：

* **已完结秒跳过**：扫描时若发现本地历史 `org_[特征码].json` 且标记为 `Uploaded: true`，**直接 0 字节网络开销秒跳过**，不再重复下载和切片。
* **断点续切自愈**：如果上次任务因中途断电或 Telegram 频控被掐断，再次运行时若本地 cache 存在大小完好的 `org_[特征码].mp4`，**直接跳过下载拉取，原地执行秒切与发射投递**。

### 🛡️ 2. 双重智能控流熔断保护器

* **普通批次「容量熔断」**：在连发多个小于 2GB 的常规视频时（例如连续 3 个 1.5GB 的原片），即使未满 `-n` 设定的文件个数，只要当前组的**累积总体积即将突破 2.0 GB**，代码会自动触发容量熔断保护，提前打包发射。**彻底杜绝了因单次 Multipart 请求包过大导致本地 Bot 服务器超时、掐断连接（Connection Reset）的通病。**
* **大视频切片「自适应二次分批」**：当巨型视频被裂变切出 10 个以上的分段短视频碎片时，系统会自动对其进行二次分组。以 `-n=10` 为横切面，自动拆分成`相册子集 A (1-10)` ➜ `休眠` ➜ `相册子集 B (11-15)` 分批平滑过渡，**完美规避了 Telegram 官方画廊单次不能超 10 个媒体的铁律红线**。

### 🎬 3. 秒级流拷贝切片与首帧咬定

* **FFmpeg 无损下刀**：利用体积向时间的智能映射算法，精准换算出分割秒数。采用 `-c copy -reset_timestamps 1` 物理搬运。**切片过程不占本地磁盘空间、不解码、不吃 CPU、几秒内完成，且每个分段在 TG 手机端点播时进度条完美对齐、不黑屏。**
* **绝对第 0 秒首帧提取**：将 `-ss 0` 严格挪至输入 `-i` 之前，达成真正的输入流高速物理寻道。**在 0 毫秒内瞬间咬定视频的第一帧作为 TG 完美的封面缩略图**，解决了开头有几帧黑屏导致没有封面的顽疾。

## 💻 生产环境复合命令示例

```bash
# 1. 深度审计模式：以每批 7 个文件为线切分，核对包含标准4位年份的创建/修改时间以及文件大小
tgup -n=7 -t=list --sort=create /path/to/webdav/dir

# 2. 强攻大视频：强制激活大视频切片逻辑，将文件大小倒序（从最大视频开始上传），且设定批次冷却 20 秒
tgup -n=10 -s=20 --spil --sort=size_desc /path/to/webdav/dir

# 3. 混合控流生产：仅过滤转码小体积的 m4v 视频，预处理完成后正式投递并附带全局标题
tgup -type=m4v --transcode --sort=size --title='推特反差女神姬美2026最新合流 #推特 #反差' /path/to/dir

```
